# Метаданные для публикации на Хабре и в Telegram

## Хабр

### Хабы (разделы на сайте)

Минимум 2, максимум 3 — обязательны при публикации:

1. **Системное администрирование** — основной
2. **Информационная безопасность** — про VPN, обход блокировок, SSH
3. **Open source** — поскольку проект MIT

Опциональный — **DevOps** (если сочтёшь подходящим).

### Теги (ключевые слова для поиска внутри Хабра)

Вводить через запятую в поле тегов при создании статьи:

```
VPN, sing-box, Russia, Россия, VLESS, Reality, xray, split-tunneling,
Claude Code, Anthropic, Windows Service, WinSW, LaunchAgent,
Shadowrocket, iOS, macOS, open source, 3x-ui, прокси, обход блокировок
```

### Краткое описание (будет в превью в ленте Хабра, до 250 символов)

Вариант 1 (короткий, фактический):

> Собрал open-source репо из 4 Claude Skills: устанавливают sing-box VPN на Windows / macOS / iOS со split-tunneling под РФ. Чтобы Claude Code не слетал с 403, а STALCRAFT / банки на .com не банили IP. 670 ₽/мес за VPS.

Вариант 2 (более эмоциональный, с zest):

> Три платных VPN за год — каждый сломался через 1-2 месяца, 6000 ₽ в ноль. Собрал свой стек на sing-box + Claude Skills под Windows / macOS / iOS. Стабильно работает, стоит 670 ₽/мес, и никто не выкидывает меня с 403 раз в день.

Вариант 3 (инженерно-цепкий):

> sing-box как Windows Service, LaunchAgent на Mac, Shadowrocket через iCloud буфер. Split-tunneling под РФ-специфику (банки на .com идут direct). Claude-on-VPS как центральный инструмент установки. Код в MIT-репо.

**Мой выбор:** вариант 2 — самый цепкий для целевой аудитории (разработчики в РФ).

### Обложка статьи (необязательно, но желательно)

Хабр рекомендует обложку 780×440 px. Варианты:
1. Скриншот ASCII-диаграммы из секции 3 (архитектура) на тёмном фоне
2. Скриншот `Get-Service sing-box` с `Running, Automatic` — чистый «успех»
3. Mock-up структуры репо (скриншот VSCode Explorer с папками windows/macos/ios)

Если нет времени — можно без обложки, просто текст.

### Что стоит вставить в статью (картинки / код)

В markdown сейчас нет картинок, только ASCII-диаграммы. Опциональные улучшения для Хабр-версии:

1. **Секция 3 (Архитектура)** — ASCII-диаграмму оставить, это читается монотонным шрифтом
2. **Секция 5 (Windows)** — скриншот `services.msc` с запущенной службой `sing-box VPN` (1 шт)
3. **Секция 7 (iOS)** — скриншот iCloud-буфера с VLESS-ссылкой (1 шт, размытие над ключами)
4. **Секция 8 (банки на .com)** — скриншот браузера где bot.gazservis72.ru или STALCRAFT показывает «региональный бан» (1 шт, до того как настроил split-tunneling)

Всего 3-4 картинки. Если нет — статья нормально читается и без них.

## Что сделать при публикации

1. Залогиниться на habr.com (аккаунт должен быть с достаточной кармой или инвайтом — если нет, сначала получить)
2. «Написать публикацию» → «Статья»
3. Скопировать текст из `habr.md`, вставить в редактор (Хабр понимает Markdown)
4. Проверить что все ссылки работают
5. Добавить хабы (2-3)
6. Добавить теги (см. выше)
7. Добавить обложку (опционально)
8. Краткое описание в поле «Превью» (вариант 2 из блока выше)
9. **Предпросмотр** — проверить как отрисовываются таблицы и ASCII-диаграмма
10. «Опубликовать»

После публикации:
- Запостить анонс в TG (если есть канал)
- Открыть issue в AndyShaman/3x-ui-skill с предложением кросс-линка (опционально)
- Следить за комментариями первые 24 часа — на Хабре критикуют активно, отвечать конструктивно
